## Описание проекта
Программный модуль предназначен для автоматизации сбора, обработки и визуализации ключевых метрик подбора персонала (HR-аналитики). Система извлекает транзакционные логи переходов кандидатов по этапам отбора из внешней СУБД, выполняет их очистку и агрегацию (ETL), после чего строит интерактивные аналитические дашборды.

### Основной функционал:
* Визуализация воронки найма с расчетом конверсии между этапами.
* Мониторинг соблюдения регламентных сроков (SLA) на каждом этапе отбора.
* Расчет метрики времени закрытия вакансий (Time-to-fill).

---

## Архитектура системы
Проект реализован в рамках классической трехзвенной клиент-серверной архитектуры:
1. **Presentation Layer (Клиентская часть):** SPA-приложение на React + TypeScript + Tailwind CSS.
2. **Business Logic Layer (Серверная часть):** API-сервер на Node.js (Express), осуществляющий ETL-процесс (извлечение, преобразование и отдачу агрегированных данных).
3. **Data Layer (База данных):** Реляционная СУБД MySQL. Хранение истории переходов организовано в таблице `Application_History` в третьей нормальной форме (3NF).

## Математический аппарат и метрики

### 1. Конверсия этапов воронки ($C_i$)
Определяет долю кандидатов, успешно перешедших на текущий этап $i$ относительно предыдущего этапа $i-1$:
$$C_i = \frac{N_{i}}{N_{i-1}} \times 100\%$$
Где $N_i$ — количество кандидатов на текущем этапе, $N_{i-1}$ — количество кандидатов на предыдущем этапе.

### 2. Время закрытия вакансии (Time-to-fill, $TTF$)
Рассчитывается как разность между датой закрытия вакансии ($T_{close}$) и датой её открытия ($T_{open}$):
$$TTF = T_{close} - T_{open}$$
Для пула вакансий рассчитывается среднее значение:
$$\overline{TTF} = \frac{1}{M}\sum_{j=1}^{M} TTF_j$$
Где $M$ — количество закрытых вакансий.

### 3. Показатель соблюдения SLA ($SLA_{rate}$)
Определяет долю переходов, выполненных без нарушения регламентного времени:
$$SLA_{rate} = \frac{A_{on\_time}}{A_{total}} \times 100\%$$
Где $A_{on\_time}$ — число переходов по этапам, уложившихся в норматив, а $A_{total}$ — общее число переходов.

### Способ Б. Запуск модуля локально на вашем компьютере
Если вам необходимо развернуть и запустить проект локально из этого репозитория:

#### Экспресс-вариант (для Windows и macOS/Linux):
В корень проекта добавлены автоматические скрипты инициализации и запуска:
* **Для Windows:** Просто дважды кликните по файлу `setup.bat`. Скрипт сам проверит Node.js, создаст файл `.env`, обновит зависимости и откроет дашборд в браузере.
* **Для macOS/Linux:** Запустите скрипт `setup.sh` в терминале (предварительно сделав его исполняемым: `chmod +x setup.sh && ./setup.sh`).

#### Альтернативный ручной вариант:
1. **Установите требования:** На компьютере должен быть установлен [Node.js](https://nodejs.org/) (версии 18 или выше).
2. **Клонируйте репозиторий и перейдите в его папку:**
   ```bash
   git clone https://github.com/BelfiGorchik/-Recruitment-Funnel-Analyzer-.git
   cd -Recruitment-Funnel-Analyzer-
   ```
3. **Установите зависимости:**
   ```bash
   npm install
   ```
4. **Запустите модуль в демонстрационном режиме:**
   ```bash
   npm run dev
   ```
